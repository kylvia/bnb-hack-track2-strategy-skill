# Submission Checklist

Do not submit externally until the owner explicitly approves.

## Ready

- [x] Strategy spec generator runs locally.
- [x] CMC-compatible quote fixture included.
- [x] Optional CMC live fetcher refuses to run without `CMC_API_KEY`.
- [x] Backtest harness runs locally.
- [x] Sample report includes a losing period and drawdown.
- [x] Demo transcript can be generated.
- [x] Submission-form draft exists.
- [x] Judging map exists.

## Still Needed Before External Submission

- [x] Owner approves external submission.
- [x] Owner approves whether to use a real `CMC_API_KEY`.
- [x] If key is provided, regenerate report from live/sponsor data.
- [x] Final review for no profit guarantees and no financial advice.
- [x] Final review for no private wallet/account/address leakage.
- [ ] Submit through the official BNB HACK flow manually or with explicit
      owner-approved browser action.
- [x] If DoraHacks requires a public repository URL, approve or create a GitHub
      repo from this directory.

## Hard Stops

- Wallet connection.
- Signature.
- Token approval.
- Swap, bridge, perp, staking, lending, borrowing, or transfer.
- Private key, seed phrase, cookie, password, 2FA, or account session request.
- External submission without owner approval.
